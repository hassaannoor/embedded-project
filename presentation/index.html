The "Hardware" Difference

The Concept

A Flappy Bird clone where voice pitch controls height. No buttons, just physics and sound.

The Unique Challenge

  • Unlike pure software, we have Real-Time Hard Deadlines
  • VGA: Requires microsecond-perfect timing (31.25 kHz)
  • Audio: Sampling at 125 kHz simultaneously
All running on a single-threaded 8-bit Arduino Uno with Shared Memory Concurrency

Specifications

The Contract of "Noise"

Topic: Preconditions and Postconditions

The Problem: How do you specify "human voice"?

Strong Precondition

"Input must be a pure Sine Wave."

  • Easier to code
  • Impossible for users

Weak Precondition

"Input can be any noisy signal."

  • Harder to code
  • Robust for users

Our Choice

Weakened the precondition to accept noise but enforced a Postcondition that the output is a stabilized frequency or a specific error code.

Mutability

The Screen Tearing War

Topic: Mutability & Aliasing

The Invariant: The screen buffer PXL_DATA must not change while it is being drawn to the monitor.

The Violation:

  • The Game Logic (Main Loop) tries to move the bird
  • The VGA Driver (Interrupt) tries to draw the bird
  • Result: "Screen Tearing" or "Ghosting"

Solution: Confined Mutation

Only the VGA module can touch the pixel array; the Game Logic sends commands via a strict API.

Animation or diagram showing "Scanline Racing": A red line (the renderer) chasing a blue line (the game update) down the screen

Recursion

Why We Avoided It

Topic: Recursive Data Types & Safety

We have 2 KB of RAM. Stack overflow = System Crash.

Analysis:

  • Signal processing often uses recursion (Divide & Conquer FFT)
  • Bad Recursion: Overlapping subproblems (calculating the same frequency window twice) leads to O(2n) time

Our Approach

We replaced recursion with Iterative Loops for the audio pipeline to guarantee O(n) performance and constant stack usage.

Stack Usage Graph: One bar showing "Recursive Approach" hitting the ceiling (Crash), and a flat line showing "Iterative Approach" (Safe)

Concurrency

The Race for 16 Bits

Topic: Race Conditions & Shared Memory

The Bug:

The detectedFreq variable is 16-bits. The processor is 8-bit.

  • Scenario: The Audio Interrupt updates the Low Byte
  • Interrupt: The Main Loop reads the value before the High Byte is updated
  • Result: The bird teleports randomly (Data Corruption)

The Fix: Atomic Blocks

We disable interrupts cli() for the 2 microseconds it takes to read the variable.

Testing

You Can't Unit Test a Whistle

Strategy: Input Space Partitioning
